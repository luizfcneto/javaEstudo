Spring Batch - Estratégia de processamento normal em janela:
	É uma estratégia de processamento para execução de processos batch simples em uma janela batch separada onde os dados atualizados não são exigidos por usuários online ou outros processos batch, concorrência não é um problema e um único commit pode ser realizado no final da execução do batch.

	Na maioria dos casos, uma abordagem mais robusta é mais apropriada. Mantenha em mente que sistemas Batch tem uma tendencia em crescer bastante de acordo com o tempo, tanto em termos de complexidade quanto de volume de dados que são manipulados por ele. Se nenhuma estratégia de locking ( travamento ) é colocada e o sistema continuar a depender de um ponto único de commit, modificar programas Batch poderá ser doloroso. Portanto, mesmo com um simples sitema Batch, considere a necessidade de uma logica de commit para opções de reinício-recuperação ( restart-recovery ) assim como informações sobre os casos mais complexos descritos posteriormente.
